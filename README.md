# devops-cicd-github-actions
# DevOps CI/CD Pipeline using GitHub Actions and Docker

## Project Overview

This project demonstrates a complete DevOps Continuous Integration and Continuous Deployment (CI/CD) pipeline using GitHub Actions and Docker. The objective of the project is to automate the build and deployment workflow for an application, reducing manual processes and improving reliability.
The pipeline is triggered automatically whenever code changes are pushed to the repository. GitHub Actions executes the CI/CD workflow to build a Docker image and prepare the application for deployment.

This project reflects real-world DevOps practices used by modern engineering teams to improve software delivery speed, consistency, and scalability.

---
## Architecture
Developer pushes code → GitHub repository → GitHub Actions pipeline runs → Docker image is built → Application ready for deployment
CI/CD Pipeline Flow:
1. Developer pushes code to GitHub
2. GitHub Actions workflow is triggered
3. Dependencies are installed
4. Docker image is built
5. Application build process completes successfully

---

## Technologies Used
- GitHub Actions (CI/CD automation)
- Docker (containerization)
- Git (version control)
- YAML (pipeline configuration)
- Linux environment
